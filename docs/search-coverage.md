# تغطية البحث — رادار المقاولات

هذه الوثيقة تحدد استراتيجية البحث وجمع البيانات للمشروع بعد توسيع نطاقه من «منافسات اعتماد» إلى رادار يغطي دورة الفرصة كاملة: من الإشارة المبكرة، إلى التأهيل، إلى الطرح، إلى الترسية، ثم تحليل السوق والمنافسين.

## 1) طبقات البحث

### أ. فرص قابلة للتنفيذ الآن

الأولوية الأعلى لأي مصدر ينشر سجلًا يمكن ربطه مباشرة بفرصة أو منافسة حالية:

- منصة مقاول — المشاريع.
- اعتماد — المنافسات والترسيات عند تفعيل Open Data API الرسمي.
- المسار الرياضي — المنافسات العامة عند ظهورها للعامة.
- الشركة الوطنية للمياه — إعلانات المناقصات والترسيات بعد التسجيل أو عند توفر صفحات عامة.
- أي بوابة مشروع كبرى تنشر Public Tender / RFQ / RFP دون تسجيل.

الحقول المستهدفة: اسم الفرصة، الجهة، رقم المرجع، النشاط، المنطقة، تاريخ النشر، آخر موعد، الحالة، القيمة التقديرية، رابط المصدر.

## 2) خط المشاريع قبل المناقصة

هذه الطبقة تعطي إنذارًا مبكرًا قبل تحول المشروع إلى مناقصة تقليدية:

- المنافسات المستقبلية في اعتماد.
- المركز الوطني للتخصيص NCP وبوابة المستثمرين.
- الشركة السعودية لشراكات المياه SWPC ومراحل EOI/RFQ/RFP.
- PIF Private Sector Hub — Explore Opportunities.
- خطوط المشاريع لدى شركات التطوير الكبرى ومبادرات تأهيل المقاولين.

الحقول المستهدفة: اسم المشروع، القطاع، المرحلة، نوع النموذج التعاقدي، الجهة المالكة، المنطقة، حجم الاستثمار أو الطلب، تاريخ الإطلاق المتوقع، مرحلة EOI/RFQ/RFP.

## 4) إشارات النشاط الإنشائي

هذه الطبقة لا تعطي مناقصة، لكنها تكشف أين يتحرك السوق:

- بيانات الرخص الإنشائية والتجارية من بلدي ووزارة البلديات والإسكان.
- الخرائط الحرارية للرخص.
- مؤشرات عدد رخص البناء من الهيئة العامة للإحصاء.
- مؤشر تكاليف البناء CCI.
- بيانات الأنشطة والسجلات التجارية من وزارة التجارة.

الاستخدامات:

- ترتيب المناطق حسب تسارع النشاط.
- اكتشاف المدن التي يرتفع فيها إصدار الرخص قبل زيادة حجم الأعمال.
- مقارنة نمو الرخص مع عدد الفرص والمناقصات.
- دعم تقدير أسعار المشاريع وربحية العطاءات.

## 5) ملفات الشركات والمنافسين

مصادر بناء ملف المنافس:

- دليل المقاولين في منصة مقاول.
- وزارة التجارة والبيانات المفتوحة.
- نتائج الترسيات من اعتماد عند تفعيل المصدر.
- أسماء الشركات المتقدمة أو المتأهلة المنشورة في EOI/RFQ/RFP لدى NCP وSWPC والجهات القطاعية.

المؤشرات المستهدفة:

- عدد مرات الظهور.
- عدد مرات الفوز.
- إجمالي قيمة العقود.
- الجهات التي يتكرر الفوز لديها.
- المناطق والأنشطة الأقوى لكل شركة.
- معدل الفوز عند توفر بيانات المشاركات الخاسرة.

## 6) أولويات الأتمتة التالية

### أولوية 1 — موصلات عامة مباشرة

1. اعتماد Open Data API بعد توفير BASE_URL وgroupId الرسميين.
2. منصة بلدي/وزارة البلديات للرخص المفتوحة إذا توفر Endpoint أو تنزيل آلي مستقر.
3. SWPC — استخراج المشاريع المستقبلية ومراحل EOI/RFQ/RFP من الصفحات العامة.
4. NCP — استخراج المشاريع العامة ومراحلها من بوابة المستثمرين والصفحات العامة.
5. أي قائمة منافسات عامة ثابتة في بوابات المشاريع الكبرى.

### أولوية 2 — إشارات السوق

1. GASTAT CCI.
2. عدد رخص البناء الشهري.
3. بيانات الشركات والأنشطة التجارية.
4. PIF Explore Opportunities وحجم الطلب المتوقع.

### أولوية 3 — مصادر تتطلب حسابًا

لا يتم تجاوز تسجيل الدخول أو الحماية. يتم التعامل معها عبر:

- ربط رسمي عندما توفر الجهة API أو export.
- إدخال يدوي/CSV للمستخدم المخول.
- تذكير المستخدم بالتسجيل والتأهيل.
- حفظ حالة التسجيل والتأهيل داخل الرادار بدل محاولة سحب محتوى محمي.

## 7) قاعدة الجودة

كل سجل يجب أن يحمل:

- `sourceExternalId` ثابتًا.
- `sourceUrl` مباشرًا.
- اسم المصدر.
- وقت آخر تحديث.
- نوع المصدر: حي / عام / تسجيل / إعداد رسمي.
- مستوى الثقة وحداثة البيانات في الإصدارات القادمة.

لا يتم دمج سجلين من مصدرين مختلفين إلا بعد مطابقة واضحة للاسم والجهة والمنطقة والرقم المرجعي. الأولوية دائمًا للمصدر الرسمي الأحدث.
